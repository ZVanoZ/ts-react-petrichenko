# Project Brief

- **Проект**: ts-react-petrichenko
- **Платформа**: Linux (bash)
- **Преференции пользователя**:
  - Строгие типы, где возможно.
  - Использование ООП, где уместно.
  - Следование лучшим практикам и паттернам для TypeScript/React и текущего проекта.
  - Ответы ассистента — на русском.

### Назначение репозитория

- Конспект и практические задания по курсу **«Полный курс по Typescript + React»** (автор: *Иван Петриченко*).
- Репозиторий структурирован по разделам курса:
  - `10-intro` — вводный раздел.
  - `20-ts` — база по TypeScript (subdirs: `10-base`, `20-necessery`, `30-generics-manipulation`, `40-classes`, `50-decorators-and-configs`).
  - `30-practice` — раздел **React + TypeScript. Большая практика**.
- В README зафиксированы ссылки на уроки и на репозиторий с «эталонными» ответами.

### Выводы по актуальности (с MCP Context7)

- По данным официальной документации TypeScript (`/websites/typescriptlang`), актуальные версии находятся в ветке 5.x (v5.8–5.9). Базовые темы курса (примитивы, функции, массивы, объекты, union, generics, классы) **остаются актуальными** — они являются фундаментом языка.
- Новые возможности TS (доп. utility-типы, улучшения в control flow, некоторые изменения конфигурации) в курсе могут не освещаться, но это не делает базовый материал устаревшим, а лишь неполным относительно последнего стандарта.
- По React (`/reactjs/react.dev`) актуальны концепции серверных компонентов, `use` для promises, улучшенный `Suspense`, concurrent-фичи и более продвинутый роутинг. Если курс ориентирован на классический SPA-клиент и базовые hooks, он **остаётся полезным**, но не покрывает современные паттерны (Server Components, продвинутые переходы, новые подходы к роутингу).

Кратко: репозиторий даёт актуальную базу по TypeScript и React, но не охватывает продвинутые возможности последних релизов (TS 5.x, новые фичи React 18+).
