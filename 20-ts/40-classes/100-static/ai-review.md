## Отчёт об актуальности урока «Статичные свойства и методы»

```text
Дата проверки: 2026-03-16
Статус: Актуально
```

### Краткий вывод

Урок остаётся **актуальным** для современных версий TypeScript: синтаксис `static`‑свойств и методов, модификаторы доступа, запрет «статических классов» и приём с закрытым/защищённым конструктором по‑прежнему используются. Статические блоки инициализации также соответствуют текущей спецификации JS/TS.

### Проверенные темы и замечания

- **Статические свойства и методы классов**
  - Синтаксис объявления `static myProp: string = '...'` и вызовов `Demo.myProp` / `Demo.myMethod()` полностью соответствует актуальной документации TS.
  - Использование модификаторов доступа (`public` по умолчанию, `protected`, `private`) для `static`‑свойств и методов поддерживается и ведёт себя так же, как и для нестатических членов.

- **Отсутствие «статических классов» в TS**
  - Указание на ошибку `TS1184: Modifiers cannot appear here` при попытке написать `static class Demo {}` остаётся корректным: TypeScript по‑прежнему не поддерживает модификатор `static` у объявления самого класса.
  - Рекомендация использовать закрытый/защищённый конструктор для имитации «статического» класса считается идиоматичным приёмом.

- **Доступ к `static` в иерархии наследования**
  - Примеры с `Demo` / `DemoChild` и чтением `Demo.myStaticProperty` и `DemoChild.myStaticProperty` отражают актуальное поведение: статические члены связаны с конструктором класса, а не с экземплярами.

- **Статические блоки инициализации (`static { ... }`)**
  - Конструкция `static { ... }` в классе, выполняющаяся один раз при загрузке модуля/скрипта, поддерживается в современных версиях TypeScript и соответствует текущей документации.
  - Демонстрация изменения `Demo.myStaticProperty` внутри статического блока и логирования до/после изменения соответствует практическому использованию.

### Что можно добавить для полноты (не критично)

- Кратко показать, как `static`‑члены ведут себя в контексте:
  - generics‑классов (ограничения на обращение к параметрам типа из статических членов);
  - `abstract`‑классов и `static`‑методов, переопределяемых в наследниках.
- Упомянуть, что:
  - статические поля **не попадают** в экземпляры объектов и не участвуют в проверке типов при работе с экземплярами;
  - статические блоки могут использоваться для сложной инициализации, но их не стоит перегружать логикой из соображений читаемости.
