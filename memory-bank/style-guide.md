# Style Guide

### TypeScript

- Включать строгие типы:
  - избегать `any` и неявных `any`;
  - использовать `unknown` вместо `any`, когда тип ещё нужно уточнить;
  - по возможности описывать сложные структуры через отдельные `type`/`interface`.
- Использовать говорящие имена типов (`UserDto`, `OrderId`, `FetchPostsResponse` и т.п.).

### React + TS (для раздела практики)

- Компоненты и hooks:
  - функциональные компоненты с типизацией пропсов через `interface Props`;
  - hooks с явными обобщёнными параметрами, если это повышает читаемость.
- Стейт/props:
  - минимизировать `any` в пропсах и стейте;
  - выделять общие типы в отдельные файлы (например, `types.ts`) по разделам.

### Документация и заметки

- В `readme.md` по урокам:
  - фиксировать только ключевые концепции и нестандартные моменты;
  - давать ссылки на официальную документацию (TS/React) вместо копирования больших кусков теории.

### Экономия токенов

- При обращении к AI:
  - указывать конкретный путь к файлу/директории вместо копирования больших блоков кода;
  - формулировать задачи кратко и чётко (1–3 предложения), а детали — по мере уточнения.
